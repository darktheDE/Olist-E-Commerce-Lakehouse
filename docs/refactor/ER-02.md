## Phân tích nguyên nhân gốc rễ (RCA)

**Log lỗi chính:** 
- `sqlite3.OperationalError: no such table: logs`
- `Failed to add user to db session: Class 'werkzeug.local.LocalProxy' is not mapped`

**Nguyên nhân:**
- Chú ý vào `sqlite3` và bảng `logs`. Cơ sở dữ liệu mặc định của Apache Superset dùng cho lưu trữ app metadata, role, user, dashboard (hiện tại đang là SQLite nằm trong container) bị rỗng – chưa được chạy lệnh định hình cấu trúc dữ liệu (`superset db upgrade`). Do cấu trúc bảng bị thiếu, khi Superset khởi chạy và có event truy cập log vào, nó sẽ văng lỗi vì không thấy bảng `logs` để insert sự kiện.
- Trong `docker-compose.yml`, service `superset` chỉ thiết lập chạy cài đặt một vài thư viện phụ thuộc của Spark, và gọi thẳng`/usr/bin/run-server.sh` để đẩy con Server Web lên thay vì kích hoạt tiến trình nạp Init DB (Schema Upgrade) - thứ vốn bắt buộc trong lần boot đầu tiên.
- Hiện có sẵn đoạn mã lệnh `/app/scripts/setup-superset.sh` trong hệ thống (gắn qua volume `./scripts:/app/scripts`), chứa toàn bộ logic tạo admin, upgrade DB, tạo roles, và đăng ký database Spark theo chuẩn. Lỗi ở đây đơn giản là script trên bị bỏ quên (không một service nào trigger nó gọi qua).

## Kế hoạch triển khai (Resolution Plan)

> **Trạng thái:** � Đã triển khai (Chờ nghiệm thu)

**Mục tiêu:** Thực thi kịch bản Init của Superset (`setup-superset.sh`) nhằm sinh ra đầy đủ Schema (bao gồm bảng logs, users), cấu hình Admin Role, và liên kết tới Thrift Server mà không làm sập lại hệ thống.

**Kế hoạch:**
Có 2 cách tự động hoá hoặc xử lý tay. Tôi đề xuất cập nhật theo cách **cấu hình Docker Compose service mới `superset-init`** (gọn gàng, xịn như `airflow-init` và triệt để cho những lần `docker compose down -v` sau này trên production).

**Các bước triển khai trên codebase:**
1. **Tinh chỉnh File `scripts/setup-superset.sh`**:
   - Thêm đường dẫn trỏ tới env ảo (`/app/.venv/bin/superset`) khi chạy các câu lệnh, vì file compose cài lib dependency `.venv` nên ta dùng đường dẫn tuyệt đối cho an toàn.
2. **Sửa `docker-compose.yml`**:
   - Tạo thêm service tạm tên là `superset-init`, dùng chính base image của `superset` và chung các biến / volume mount, `depends_on: spark-thrift-server`. 
   - Lệnh khởi động service này sẽ thay bằng `bash /app/scripts/setup-superset.sh`. 
   - Đặt bổ sung `depends_on: superset-init (service_completed_successfully)` vào service Web `superset` chính hiện tại, đảm bảo web UI chỉ start khi toàn bộ Roles và Meta đã dựng xong.
3. **Nghiệm thu (Verification)**:
   - `docker compose build / up -d`.
   - Chờ hệ thống boot, kiểm tra Superset log bằng lệnh `docker compose logs superset`, log không được văng lỗi `no such table: logs` nữa.
   - Kiểm thử truy cập UI (cổng 8088), username/password dùng từ env (`SUPERSET_ADMIN/SUPERSET_PASSWORD`).
