## RF-01 — Sửa lỗi Hive Metastore khởi động lần đầu

> **Trạng thái:** 🟡 Đã triển khai — Chờ manual test

### Phân tích chi tiết

**Vấn đề thực sự:** PostgreSQL database `metastore` trống hoàn toàn khi lần đầu khởi động. Spark cần schema Hive (≥ 74 tables như `DBS`, `SDS`, `TBLS`, `VERSION`, ...) trước khi có thể hoạt động.

**Tool đúng đắn:** `schematool` — công cụ CLI đi kèm Apache Hive (và được nhúng sẵn trong Spark) để init/validate/upgrade schema Hive Metastore.

Lệnh cần chạy:
```bash
/opt/spark/bin/schematool -dbType postgres -initSchema -verbose
```

**Cách tích hợp an toàn:** Tạo thêm 1 service `hive-metastore-init` trong `docker-compose.yml`, dùng cùng image `apache/spark`, chạy `schematool` một lần duy nhất, sau đó exit. Các service khác (`spark-thrift-server`, `airflow-*`) phải `depends_on: hive-metastore-init: condition: service_completed_successfully`.

**Thứ tự khởi động đúng sau refactor:**
```
postgres (healthy)
    └─> hive-metastore-init (completed_successfully)
            └─> spark-thrift-server
            └─> airflow-init
                    └─> airflow-webserver
                    └─> airflow-scheduler
```

**Xung đột cần giải quyết:** Sau khi schema đã được init, cần đảm bảo:
- `spark.hadoop.datanucleus.schema.autoCreateAll = false` (không tự tạo lại)
- `spark.hadoop.hive.metastore.schema.verification = true` (verify đúng version)
- Cả `spark-defaults.conf` lẫn `src/spark_session.py` phải đồng nhất

### Các file sẽ thay đổi

| File | Thay đổi |
|------|----------|
| `docker-compose.yml` | Thêm service `hive-metastore-init`; cập nhật `depends_on` cho `spark-thrift-server` và `airflow-common` |
| `conf/spark-defaults.conf` | Đồng bộ `autoCreateAll=false` + `verification=true` |
| `src/spark_session.py` | Đồng bộ `autoCreateAll=false` + `verification=true` |

### Kế hoạch triển khai

**Bước 1:** Thêm service `hive-metastore-init` vào `docker-compose.yml`

Service này sẽ:
1. Dùng image `apache/spark:3.5.0` (cùng version với thrift-server)
2. Mount `./jars` và `./conf/spark-defaults.conf`
3. Chạy lệnh: init schema PostgreSQL bằng `schematool`
4. `restart: "no"` — chỉ chạy 1 lần

**Bước 2:** Cập nhật `depends_on` của `spark-thrift-server`

Thêm điều kiện:
```yaml
hive-metastore-init:
  condition: service_completed_successfully
```

**Bước 3:** Cập nhật `depends_on` của `airflow-common` anchor

Thêm điều kiện tương tự để các Airflow workers không chạy Spark job trước khi Hive schema sẵn sàng.

**Bước 4:** Đồng bộ config Spark

`conf/spark-defaults.conf` và `src/spark_session.py` cùng dùng:
```
autoCreateAll = false
schema.verification = true  (hoặc false nếu dùng schematool đã init)
```

> ⚠️ **Lưu ý quan trọng:** Sau khi `schematool` đã init schema thành công, giá trị đúng là:
> - `autoCreateAll = false` — đã có schema rồi, không cần auto-create
> - `verification = true` — Spark sẽ verify version schema khớp với Hive version

### Hướng dẫn Manual Test RF-01

Sau khi triển khai xong, thực hiện các bước sau để verify:

**Test 1 — Clean start (lần đầu chạy):**
```bash
# Xóa sạch volume PostgreSQL để test fresh
docker compose down -v

# Khởi động lại từ đầu
docker compose up -d

# Theo dõi log của hive-metastore-init
docker compose logs -f hive-metastore-init
```
✅ **Pass khi:** Log in ra `schemaTool completed` và service exit với code 0.

**Test 2 — Verify schema đã được tạo:**
```bash
docker exec -it olist-postgres psql -U metastore -d metastore -c "\dt"
```
✅ **Pass khi:** Thấy danh sách các bảng Hive như `DBS`, `SDS`, `TBLS`, `VERSION`, ...

**Test 3 — Thrift Server khởi động sạch:**
```bash
docker compose logs spark-thrift-server | grep -E "ERROR|Started|listening"
```
✅ **Pass khi:** Không có dòng `MissingTableException`, thấy `ThriftBinaryCLIService listening on 0.0.0.0/0.0.0.0:10000`.

**Test 4 — Spark Job kết nối được Hive:**
```bash
docker exec -it olist-airflow-webserver bash -c "python /opt/airflow/jobs/register_tables.py"
```
✅ **Pass khi:** Log in ra `[SUCCESS] All Bronze Delta tables have been successfully registered.` (hoặc không báo lỗi Metastore).
