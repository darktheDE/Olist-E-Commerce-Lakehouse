## Phân tích nguyên nhân gốc rễ (RCA)

Lỗi nhận được: `MetaException(message:One or more instances could not be made persistent)`.

**Nguyên nhân:**
1. Lỗi này sinh ra trong quá trình Spark (ở `register_tables.py`) cố gắng ghi metadata vào Hive Backend (Postgres) nhưng các bảng bảo mật/quyền hoặc khóa (như `TBL_PRIVS`) không tồn tại, hoặc bị Deadlock.
2. Quá trình `RF-01` trước đó thực hiện chiến lược dùng `init-hive-schema.py` ép DataNucleus tự động tạo schema (`autoCreateAll=true`). Tuy nhiên, DataNucleus tự sinh bảng thường bị khuyết các bảng chuyên biệt của Hive 2.3.0 hoặc dính lỗi transaction lock deadlock như tài liệu `BDANGR20-23.md` đã cảnh báo.
3. Không thể dùng command `/opt/spark/bin/schematool` bởi public image `apache/spark` và `jupyter/pyspark-notebook` không chứa sẵn tool này.

**Giải pháp:**
Vì không có `schematool` và `autoCreateAll` không triệt để, chúng ta bắt buộc làm theo đúng kinh nghiệm trong `BDANGR20-23.md`:
> *"Cách xử lý là chạy script chuẩn của Hive (`hive-schema-2.3.0.postgres.sql`) trực tiếp vào Postgres để build Schema trước, thay vì dựa vào auto-create của DataNucleus."*

## Kế hoạch triển khai (Resolution Plan)

> **Trạng thái:** � Đã triển khai (Chờ nghiệm thu)

**Mục tiêu:** Xử lý triệt để Hive schema, loại bỏ hoàn toàn sự phụ thuộc vào DataNucleus auto-create, giúp các Spark tasks đăng ký bảng Delta thành công.

**Các bước thực hiện:**
1. **Tải script SQL chuẩn:** Tải file DDL `hive-schema-2.3.0.postgres.sql` của Apache Hive về thư mục `scripts/`.
2. **Cập nhật Docker Compose (`docker-compose.yml`):**
   - Loại bỏ service `hive-metastore-init` vì chạy bằng Spark/DataNucleus đang gây lỗi.
   - Thêm `scripts/hive-schema-2.3.0.postgres.sql` vào thư mục mount `/docker-entrypoint-initdb.d/` của service `postgres`. Nhờ đó, Postgres ở lần start đầu tiên (khi volume trống) sẽ tự động sinh full bảng cho Metastore mà không cần Spark nhúng tay vào.
3. **Dọn dẹp Cleanup:** Xóa file script `scripts/init-hive-schema.py` đã tạo từ RF-01.
4. **Chuẩn hóa biến cấu hình Spark:** Tại các file `conf/spark-defaults.conf`, `src/spark_session.py`, và các nơi khác:
   - Đảm bảo `spark.hadoop.datanucleus.schema.autoCreateAll=false`
   - Đảm bảo `spark.hadoop.hive.metastore.schema.verification=false` (hoặc `true` sau khi đã chèn thành công script SQL). Tốt nhất tạm gán `false` để pass kiểm tra khắt khe.
5. **Nghiệm thu (Verification):**
   - `docker compose down -v` (xóa volume cũ để reset Postgres).
   - `docker compose up -d`.
   - Vào giao diện Airflow trigger lại task `register_tables` để kiểm tra.
